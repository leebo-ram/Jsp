서블릿(Servlet)
웹프로그래밍에서 클라이언트의 요청을 처리하고 그 결과를 다시 클라이언트에게 
전송하는 Servlet 클래스의 구현 규칙을 지킨 자바 프로그래밍 기술을 뜻합니다.

특징
1. 자바를 사용하여 웹 페이지를 동적으로 생성하는 서버 측 프로그램입니다.
2. java 확장자로 자바를 이용하여 웹 어플리케이션을 만들기 위해 제공되는 기술입니다.
3. 클라이언트 요청에 동적으로 작동하고, 응답은 HTML을 이용합니다.
4. MVC 패턴에서 Controller로 이용됩니다.

서블릿의 장점
1. Java를 사용해서 동적 웹 어플리케이션을 구현할 수 있습니다.
2. 플랫폼에 독립적입니다.
3. 컨텐츠와 비즈니스 로직을 분리 할 수 있습니다.
4. 유지 보수와 기능 확장이 용이합니다.

서블릿의 단점
1. 화면에 표현될 HTML코드를 프로그램적으로 작성해야합니다.
2. 서비스하기 전에 반드시 컴파일을 해야합니다.

JSP 내장 객체

request 객체
웹 브라우저의 요청 정보를 저장하고 있는 객체입니다.

메소드
getHeader() : 요청 정보의 헤더정보를 반환합니다.
getMethod() : 요청 정보의 HTTP Method를 반환합니다.
getParameter() : 요청 정보의 이름으로 요청 값을 반환합니다.
setCharacterEncoding() : 요청 정보의 인코딩을 설정합니다.
getParameterValues() : 요청 정보의 이름으로 요청 값을 배열로 반환합니다.


<input type="text" name="userid">	------------------> String id = request.getParameter("userid");
		         -------			      ---			  ------
						value가 저장

response 객체
웹 브라우저의 요청에 대한 응답 정보를 저장하고 있는 객체입니다.


쿠키(cookie)
1. 클라이언트에 사용자 정보를 저장합니다.
2. 클라이언트에 저장하므로 서버의 부하를 줄일 수 있지만 보안상 위험합니다.
3. 데이터의 저장이 1.2MB로 제한되어 있습니다.
//! 장바구니 담겨있는 상품들이 저장되어 있는 경우에 사용하는 기능이 쿠키이다.
//! 사이트를 이용하는데 있어서 편리한 이용을 위해 만든 기능


쿠키 생성 벙법
Cookie 객체명 = new Cookie("키", "값");

쿠키 시간을 설정하는 방법
setMaxAge() : 쿠키가 유지되는 시간을 설정합니다. (초)

60s * 60m * 24h * 30d  -> 30일

쿠키를 클라이언트에게 전달하는 방법
response.addCookie(쿠키객체);

쿠키를 서버에서 읽어오는 방법
Cookie[] 배열명 = requset.getCookies();
