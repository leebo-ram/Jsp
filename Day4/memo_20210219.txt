자바스크립트 - 비동기 서버연결하는 

Ajax(Asynchronous JavaScript And XML)
빠르게 동작하는 웹페이지를 만들기 위한 개발기법입니다. 웹 페이지 전체를 다시 
로딩하지 않고 웹 페이지의 일부분만을 갱신할 수 있습니다. 백그라운드 영역에서 
서버와 통신하여 그 결과를 웹페이지에 표시할 수 있습니다. 
서버와 데이터를 주고 받는 데이터는 JSON, XML, HTML, 텍스트 파일등이 있습니다.

Ajax 장점
1. 웹페이지 전체를 다시 로딩하지 않고 웹페이지 일부분만을 갱신할 수 있습니다.
2. 웹페이지가 로드된 후에 서버로 데이터를 요청을 보내거나 받을 수 있습니다.
3. 다양한 UI(동적 페이지)의 구현이 가능합니다.
SPA(싱글페이지어플리케이션) 라고 실무에서 말한다.

Ajax 단점
1. 페이지 이동이 없기 때문에 히스토리가 남지 않습니다.
2. 반복적인 데이터를 요청하게 되면 느려지거나 작동하지 않습니다.
3. 서버에서 클라이언트로 먼저 요청할 수 없습니다.

XMLHttpRequest 객체
Ajax의 가장 핵심적인 구성 요소는 XMLHttpRequest 객체입니다. 
이 객체는 웹 브라우저가 서버와 데이터를 교환할 때 사용합니다. 이 객체는 백그라운드에서
계속 서버와 통신할 수 있는 기능을 제공합니다.

XMLHttpRequest 객체 생성 방법
const 상수명 = new XMLHttpRequest();

서버에 데이터를 요청하는 방법
open() : 서버에 보낼 Ajax요청의 형식을 설정합니다.
open(전달방식, URL주소, 동기여부);
전달방식 : get, post 방식 중 하나를 선택
URL 주소 : 요청을 처리할 서버의 파일 주소
동기 여부 : 요청을 동기식으로 전달할지 비동기식으로 전달할지 선택(기본 true : 비동기)

send() : 작성된 Ajax를 서버로 전달합니다.
send(); : get방식
send("문자열") : post 방식


readyState : xhr의 상태를 나타내는 프로퍼티
처음(객체 생성시) : 0
open 메소드를 호출 : 1
send 메소드를 호출 : 2
loading 이 진행되면 : 3
완료가 되면 : 4

status : xhr이 접속한 서버 페이지 상태
200 : 정상적인 호출
404 : 페이지가 없음

웹페이지는 대부분 비동기식으로 진행이됨.
